# 内容目录说明

本目录用于存放网站文章，结构会自动生成目录。

## 目录结构

- **一级条目**：在 `data/` 下创建文件夹（如 `序`、`地理志`、`年表`）
- **二级条目**：在文件夹内放置 `.md` 文件，或创建子文件夹
- **三级条目**：在子文件夹内放置 `.md` 文件

## 示例

```
data/
├── 序/
│   ├── 前言.md
│   └── 关于本大陆志.md
├── 地理志/
│   ├── 大陆概览.md
│   └── 诸王国/          ← 子文件夹 = 三级分类
│       ├── 北境.md
│       └── 南境.md
└── 年表/
    └── 第一纪元.md
```

## Markdown 文件格式

在文件顶部使用 YAML frontmatter 设置标题：

```yaml
---
title: 前言
titleEn: Preface
---

正文内容使用 Markdown 语法...
```

- `title`：中文标题（必填，也可用文件名）
- `titleEn`：英文副标题（可选）

## 自定义排序

在**任意层级**的文件夹内放入 `_order.json`，即可指定该层下条目在目录中的顺序。

- 文件名：`_order.json`（以 `_` 开头，不会出现在目录中）
- 内容：JSON 数组，按希望显示的顺序列出**文件夹名**或**文章文件名（不含 .md）**

**示例**：在 `data/` 下创建 `_order.json`：

```json
["序", "地理志", "年表"]
```

在 `data/序/` 下创建 `_order.json`：

```json
["关于本大陆志", "前言"]
```

则「序」下的顺序会变为：关于本大陆志 → 前言。

未出现在 `_order.json` 中的条目会排在后面，并按名称排序。
