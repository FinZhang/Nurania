# 诺拉尼亚（Nurania）项目说明

本文档整合项目愿景、已实现功能、数据与部署约定，便于跨对话与后续维护时对齐上下文。

---

## 1. 核心愿景与风格

- **定位**：中世纪西幻背景世界观「诺拉尼亚大陆」的展示站，设定为后世为学者塞勒内斯撰写的一系列著作合集。
- **适配**：完整支持电脑宽屏与移动端窄屏。
- **风格**：史书背景、史诗感、神秘主义、羊皮纸质感与现代极简 UI 结合（类似《艾尔登法环》/《巫师 3》的 UI）。
- **视觉**：书籍纸质背景、带手写感的字体；交互带微光（Glow）与淡入淡出（Fade）。

---

## 2. 技术栈

- **框架**：Next.js（App Router）
- **样式**：Tailwind CSS
- **动画**：Framer Motion
- **图标**：Lucide React
- **构建**：生产环境 `output: "export"` 静态导出；开发环境不导出以放宽 `generateStaticParams` 校验。

---

## 3. 路由与站点结构

- **basePath**：`/Nurania`，所有资源与链接均带此前缀（见 `next.config.ts`、`image-loader.ts`）。
- **路由约定**：
  - `/`：**书架**（主页），展示多本书封面，点击某书进入该书目录。
  - `/[bookSlug]/toc`：某书的目录页（如 `/compendium/toc`、`/book-2/toc`）。
  - `/[bookSlug]/article/[...slug]`：某书的文章页（如 `/compendium/article/作者序`）。
- **可扩展性**：新增书籍只需在 `src/lib/books.ts` 的 `BOOKS` 数组中增加一项，并在 `data/`、`public/` 下增加对应平级目录即可，无需改页面组件。

---

## 4. 书籍配置与数据/资源目录

- **配置入口**：`src/lib/books.ts`。每本书包含：
  - `slug`：路由与标识（如 `compendium`、`book-2`）
  - `title` / `titleEn`：中英文书名
  - `cover`：封面图路径（如 `/Compendium/book_cover.png`）
  - `intro`：目录页简介区块正文
  - `dataDir`：该书 Markdown 数据目录（相对项目根，如 `data/Compendium`）
  - `publicDir`：该书图片在 `public` 下的文件夹名（如 `Compendium`）
  - `footerSubline`（可选）：该书内页脚左侧副标题

- **数据目录**：每本书在 `data/` 下占一个**平级**文件夹，彼此不嵌套。
  - 例：`data/Compendium/`（第一本）、`data/book-2/`（第二本）。
  - 每书目录内为层级 Markdown 与 `_order.json`，由 `src/lib/content.ts` 按 `bookSlug` 读取对应 `dataDir`。

- **图片目录**：每本书在 `public/` 下占一个**平级**文件夹。
  - 例：`public/Compendium/`（第一本封面、世界地图、诸国列志等）、`public/book-2/`（第二本封面等）。
  - 引用时路径如 `/Compendium/book_cover.png`、`/book-2/book_cover.png`。

## 5. 页面功能与交互

### 5.1 书架（主页）

- 展示所有书的封面；宽屏两列、窄屏上下排列；封面下方不显示书名。
- 宽屏下页面高度等于视口（约 `100vh - 13rem`），不出现整页滚动；地图背景 `fixed` 铺满视口，与页脚之间无空白。
- 点击某书封面进入该书目录页 `/[bookSlug]/toc`。

### 5.2 页眉（Header）

- **在书架（`/`）**：左侧显示「塞勒内斯全集」，右侧**不显示**目录按钮。
- **在书籍内**：左侧为当前书名（可点击回到书架 `/`），右侧为「书架」链接 +「本书目录」链接（指向当前书 `/[bookSlug]/toc`）；文章页窄屏时「本书目录」同时作为打开侧栏目录的按钮。

### 5.3 页脚（Footer）

- **在书架**：左侧「Selenus: The Complete Writings」，右下两行：「Fin Zhang © {当前年份}」「Contact: shininglight441@gmail.com」；**不显示**书架/目录按钮。
- **在书籍内**：左侧为当前书的 `footerSubline`，右下同样为上述版权与联系两行；**不显示**书架/目录按钮。
- 页脚背景为不透明浅色（如 `#b8956e`），文字为深色便于辨认；主内容区底部有适当 `pb`，避免正文被页脚遮挡。

### 5.4 目录页（`/[bookSlug]/toc`）

- 「返回书架」链接到 `/`。
- 左侧为层级目录列表，右侧（宽屏）为该书封面 + 简介 + 最近更新区块；窄屏为简介与最近更新在上、目录在下。
- 目录与最近更新中的链接为 `/[bookSlug]/article/{slug}`。

### 5.5 文章页（`/[bookSlug]/article/[...slug]`）

- **宽屏**：
  - 左侧为该书目录导航（固定高度、可独立滚动）；鼠标在**文章区域**滚动时整页滚动、页眉吸顶；鼠标在**左侧导航**上滚动时仅导航栏滚动、整页不滚动。
  - 导航栏滚动条样式：透明底、淡棕色细条，悬停时加深；仅当**切换文章**（`currentSlug` 变化）时自动将当前项滚入可视区，不在鼠标移出导航时触发。
- **窄屏**：通过页眉「本书目录」打开侧栏目录，逻辑与原先一致。

#### 题图（可扩展）

- 任意文章可在正文**标题下方任意位置**插入一行题图代码，指定图片路径后即可在页面中插入题图（不限于诸国列志）。
- **写法**：`<!-- 题图 路径 -->`，路径相对**全局 public 目录**。例如：`<!-- 题图 Compendium/诸国列志/索拉瑞斯.png -->`。
- **宽屏**：题图显示在文章右上角（float-right），上边缘略低于文章标题。
- **窄屏**：题图在上、正文在下（题图占一整行，正文从下方开始）。
- 题图代码行在解析时会被移除，不会在正文中显示；图片需放在 `public/` 下对应路径。

#### 折叠块

- 在任意 `.md` 中使用下列语法，该文章会以可折叠块渲染（由 `src/lib/fold-blocks.ts` 解析、`MarkdownWithFoldBlocks` 渲染）：
  ```text
  ::: fold 标题文字
  此处为折叠块内容（支持完整 Markdown）
  :::
  ```
- **开始行**：`::: fold 标题` 或 `:::fold 标题`（行首可带空格）。**标题支持 Markdown**（如 `::: fold ### 神启学派-祷言`），会按层级用 Cinzel 字体渲染；**若标题行无格式标记**（无 `#`、`**`、`*` 等），则使用正文格式（正文字体、字重）。
- **结束行**：单独一行 **半角** `:::` 或 `::: endfold`（行首须为 `:::`；全角 `：：：` 无法识别）。
- 折叠块标题**左对齐**，与正文一致。
- **内容处理**：若块内首行为 `### 纯文本标题` 且与折叠标题的纯文本一致，渲染时会去掉该行避免重复；开头多余空行会压缩为至多一个段落间距；展开后内容区顶部仅保留一行正文高度（1.75em），避免第一行离标题过远。
- **吸顶收起**：当折叠块标题处于吸顶状态时点击收起，收起后标题会停留在吸顶位置（通过滚动微调），再随页面正常滚动。

#### 题图与折叠块共存

- 若同一篇文章既有题图代码又有折叠块，**两者都会显示**：先渲染题图（宽屏 float、窄屏在上），再渲染全文（含折叠块）。
- **重叠处理**：若题图“覆盖”第一个折叠块起始行（即第一个 `::: fold` 所在行 ≤ 题图插入行），则第一个折叠块会加上上边距（宽屏约 600px），使折叠块整体下移到题图下方，避免与题图重叠。

#### 文章内导航

- **仅宽屏显示**：当文章正文含有三级标题（`###`）时，宽屏下在页面**最右侧**显示该文章的章节导航（左侧为本书目录、中间为正文、右侧为文章内导航）；窄屏下不显示。
- **内容与样式**：从上到下列出文中所有 `###` 标题，无总标题；每条带书签图标、上下分割线，风格与站点一致；导航区域宽度随最长条目略长，不占满整行。
- **跳转**：点击条目平滑滚动到对应小节；滚动位置自动预留页眉高度及约两行正文间距，避免被吸顶页眉遮挡。链接使用当前页 pathname + hash，兼容 basePath。
- **带题图文章**：题图页会同时渲染多套 DOM（如窄屏/宽屏），id 会赋给**最后一套**（当前视口可见）的 h3，点击时优先滚动到可见元素，保证宽屏带题图时跳转正确。
- **实现要点**：`src/lib/headings.ts` 的 `extractH3Headings` 从 Markdown 按序提取所有 `###` 并生成唯一 id；`ArticleHeadingIdInjector` 在客户端挂载后为容器内 h3 按序注入 id（多套 DOM 时赋给最后 n 个），并用 MutationObserver 在折叠块展开后补注；右侧栏为三栏布局中的第三列，`sticky top-36`、`w-48`，右侧负边距压缩留白。

## 6. Markdown 与内容约定

- 文章为 Markdown，frontmatter 支持 `title`、`titleEn`。
- 目录顺序由各目录下的 `_order.json`（名称数组）控制；无则按名称排序。
- **题图**：见上文 5.5「题图（可扩展）」；路径相对全局 `public`，如 `Compendium/诸国列志/文件名.png`。
- **可折叠块**：见上文 5.5「折叠块」；示例见 `data/Compendium/大陆总览 Overview/魔法.md`、`data/Compendium/诸国列志 Chorography of the Nations/索拉瑞斯.md`。

---

## 7. 部署约定

- **站点路径**：部署在子路径 `/Nurania`（如 https://finzhang.space/Nurania/）。
- **静态导出**：生产构建为 `output: "export"`，产物在 `out/`。
- **服务器限制**：若服务器会屏蔽以 `_` 开头的目录，需将 `out/_next` 重命名为 `out/next` 并替换 HTML/JS 中的 `_next` 为 `next`（由 `scripts/rename-next-for-deploy.js` 完成）。
- **一键部署**（在项目根目录执行）：
  ```bash
  npm run deploy:push
  ```
  顺序：`npm run build` → `npm run deploy:rewrite` → 进入 `out` 执行 `git add .`、`git commit`（带日期）、`git push origin HEAD:web-release`。若无变更会跳过 commit/push。
- **手动部署**：
  ```bash
  npm run build
  npm run deploy:rewrite
  ```
  再将 `out` 目录下全部内容部署到站点的 `/Nurania` 路径（如将 `out` 推送到远程 `web-release` 分支或 FTP 上传）。

## 9. 开发者指令（沿用）

- **全局样式**：在 `globals.css` 中维护「奇幻史诗」主题色板。
- **组件化**：导航栏、页脚、内容区均为独立组件。
- **响应式**：所有页面需适配移动端与宽屏。
- **数据驱动**：文章与目录结构由 `data/{书}/` 下 Markdown 与 `_order.json` 驱动，无需在代码中写死条目。
