# 部署约定与静态导出

本文档承接 `项目说明.md` 中的部署约定部分（7），重点描述：站点如何在子路径下工作、静态导出产物如何命名/适配部署环境、以及本地预览与脚本的入口。

## 1. 站点路径（basePath）
- 站点部署在子路径 `/Nurania`。
- 路由与资源链接均需要带此前缀（例如由 `next.config.ts` 的 basePath 与相关加载器共同保证）。
- 构建启用 `trailingSlash: true`：目录页访问地址带尾部 `/`。

## 2. 静态导出产物与访问形式
- 生产构建为静态导出：`output: "export"`。
- 产物目录为 `out/`。
- 示例：目录页访问地址为 ` /Nurania/compendium/toc/ `（带尾部斜杠）。

## 3. 服务器限制与 `out/_next` 重命名
- 若服务器会屏蔽以 `_` 开头的目录：
  - 部署前将 `out/_next` 重命名为 `out/next`
  - 并把产物中的 `_next` 字符串替换为 `next`
- 由 `scripts/rename-next-for-deploy.js` 完成。

## 4. 部署脚本行为摘要
- `deploy:rewrite` 主要做两件事：
  - `_next` -> `next` 重命名与 `_next` 字符串替换
  - 将 `public/compendium`、`public/upTheRiver`、`public/deathOfDevout` 合并到 `out/`（用于兼容世界地图/封面等旧目录名场景）
- `deploy:push`（一键部署）顺序：
  - `npm run build` -> `npm run deploy:rewrite` -> 在 `out` 内完成 `git add` / `git commit`（带日期）/ `git push origin HEAD:web-release`
  - 若无变更会跳过 commit/push。

## 5. 本地预览
- `npm run preview` 会把 `out` 复制到 `preview/Nurania` 并启动静态服务。
- 访问 `http://localhost:3000/Nurania/`。
- 根路径会重定向到 `/Nurania/` 以减少 404 日志。
