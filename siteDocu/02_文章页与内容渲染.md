# 文章页与内容渲染

本文档承接 `项目说明.md` 中“文章页与内容渲染”相关部分（5.5-6）。

目标：描述文章页面在站点内的可见行为、以及内容层面的写法/语法约定；避免重复 `siteDocu/内容与 Markdown 合约.md` 中的那些“写作规则总览”（两者会做边界划分）。

## 1. 文章页（`/[bookSlug]/article/[...slug]`）布局与滚动行为
- 宽屏：
  - 三栏：左侧书目录导航（固定高度、可独立滚动）+ 中间文章区域 + 右侧文章内 `###` 导航（仅在存在 `###` 时出现）。
  - 页面滚动交互：
    - 当滚动发生在“文章区域”时：允许整页滚动，同时页眉吸顶效果生效。
    - 当滚动发生在“左侧导航”时：仅导航栏滚动，整页滚动不会被触发。
  - 左侧导航滚动条为透明底、淡棕色细条，悬停加深；仅在切换文章（`currentSlug` 变化）时自动把当前项滚入可视区。
- 窄屏：
  - 通过页眉里的「本书目录」打开侧栏目录；侧栏逻辑与原先一致。

## 2. 题图（Title Image）
- 用途：在文章正文标题下方的任意位置插入一行题图注释代码。
- 语法：`<!-- 题图 路径 -->`
- 路径：相对全局 `public` 目录。
  - 例如：`<!-- 题图 compendium/诸国列志/索拉瑞斯.webp -->`
- 渲染行为：
  - 宽屏：题图在文章右上角浮动（float-right），上边缘略低于文章标题。
  - 窄屏：题图占一整行，位于正文上方。
- 解析行为：题图注释行在渲染时会从正文中移除，不会显示为文本。

## 3. 折叠块（Fold Blocks）
- 用途：在任意 `.md` 中把一段内容渲染为可折叠块。
- 语法：
  - 起始：`::: fold 标题文字`（行首可带空格；也允许 `:::fold 标题`）
  - 结束：单独一行 `:::`（半角）或 `::: endfold`（行首须为 `:::`）
- 标题渲染：
  - 标题支持 Markdown（例如在标题行写 `### 神启学派-祷言`）。
  - 当标题行无格式标记（无 `#`、`**`、`*` 等）时，使用正文格式（字体与字重与正文一致）。
  - 标题左对齐，与正文一致。
- 内容处理要点：
  - 若块内首行为 `### 纯文本标题`，且与折叠标题的纯文本一致，则渲染时会去掉该行以避免重复。
  - 开头多余空行会被压缩为至多一个段落间距。
  - 展开后，折叠内容区顶部仅保留一行正文高度（约 `1.75em`），避免第一行距离标题过远。
- 吸顶与交互：
  - 折叠块标题吸顶状态下点击收起，会保持标题停留在吸顶位置；收起后再随页面正常滚动。

## 4. 题图与折叠块共存与重叠避让
- 同一篇文章既有题图注释又有折叠块：两者都会显示。
- 重叠规则：如果题图“覆盖”到第一个折叠块起始行（即第一个 `::: fold` 所在行不晚于题图插入行），则会给第一个折叠块额外上边距（宽屏约 600px），把折叠块整体下移，避免与题图重叠。

## 5. 文章内导航（h3：`###` 提取与跳转）
- 展示条件：仅宽屏显示；当文章正文包含三级标题（`###`）时显示。
- 导航内容：按从上到下顺序列出所有 `###` 标题（不含总标题）；每条带书签图标与上下分割线。
- 跳转行为：
  - 点击条目会平滑滚动到对应小节。
  - 滚动位置会预留页眉高度及约两行正文间距，避免被吸顶页眉遮挡。
  - 链接使用当前页 pathname + hash，兼容站点 `basePath`。
- 题图文章的兼容性：
  - 题图页会渲染多套 DOM（如宽屏/窄屏视口差异），id 会赋给“最后一套”（当前可见视口）。
  - 点击导航时优先滚动到可见元素，确保宽屏带题图时跳转正确。
- 实现要点（用于理解行为来源）：
  - `extractH3Headings`：从 Markdown 中按序提取所有 `###` 并生成唯一 id。
  - `ArticleHeadingIdInjector`：客户端挂载后为文章容器内的 h3 依次注入 id；并在折叠块展开后补注（通过观察折叠展开导致的 DOM 变化）。

## 6. Markdown 与内容合约（只保留文章页所需部分）
- 文章 Markdown：
  - frontmatter 支持 `title`、`titleEn`。
- 目录顺序：
  - 目录结构由各书 data 目录下的 `_order.json`（名称数组）控制；无则按名称排序。
- 题图与折叠块：
  - 写作规则与语法见本文件第 2/3/4 节。
